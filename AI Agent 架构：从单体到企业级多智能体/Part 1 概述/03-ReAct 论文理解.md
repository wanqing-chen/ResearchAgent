## 核心思想：推理与行动的协同
ReAct 的核心逻辑在于将 “言语推理（Reasoning）” 与 “特定任务行动（Acting）” 结合在一起。

- Reasoning (推理)：类似于 Chain-of-Thought（CoT），模型生成内心的思考过程。这有助于模型制定计划、跟踪进度并根据情况调整策略。

- Acting (行动)：模型与外部环境（如维基百科、数据库或软件工具）交互，获取实时或更准确的信息。

- 协同效应：推理帮助模型决定下一步采取什么行动；行动产生的结果（观察）又会回馈给模型，引导进一步的推理

## 为什么要用 ReAct？（面试高频问题）

- 克服 CoT 的幻觉问题：纯推理（CoT）是一个“黑盒”，模型只能依靠内部知识。当知识过时或不准确时，模型会产生事实幻觉。ReAct 通过访问外部知识库（如 Wikipedia API）来确保事实的准确性。

- 解决 Act-only 的盲目性：仅有行动的模型（如 WebGPT）通常缺乏高层逻辑规划，难以处理复杂的多步任务。ReAct 的推理步骤提供了“工作记忆”和“策略调整”能力。

- 可解释性与可诊断性：ReAct 生成的轨迹（Thought-Act-Observation）清晰地展示了模型的思考过程。人类可以直观地看到模型是因为哪一步搜索错误或哪一步推理失误导致了最终失败，这在工业界调试 Agent 时至关重要。


“ReAct 范式通过将 LLM 的推理链条与外部工具的调用闭环结合，不仅解决了纯推理导致的幻觉问题，还赋予了模型动态规划和自我修正的能力，是实现可解释、高可靠 AI Agent 的关键路径。”

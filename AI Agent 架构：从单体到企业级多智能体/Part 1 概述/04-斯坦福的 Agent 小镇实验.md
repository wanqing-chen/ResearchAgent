## 核心架构：三大支柱

### 记忆流（Memory Stream）
- 这是一个包含智能体所有经历的长期记录数据库。
- 每条记录都是一个包含自然语言描述、创建时间戳和最近访问时间戳的对象。

### 反思（Reflection）
- 智能体会定期总结记忆流中的观察结果，生成更高阶、更抽象的思想。
- 反思不仅基于原始观察，还可以基于之前的反思，形成一棵“反思树”。

### 规划（Planning）
- 智能体会制定全天的行动计划（如：几点起床、去哪里工作），并随着环境变化不断调整。
- 规划采用了“由粗到细”的递归分解方法：先定全天大纲，再细化到每小时，最后细化到分钟级别的具体动作。

### 关键机制：检索函数（Retrieval Function）
由于上下文窗口限制，Agent 无法处理所有记忆，因此需要一个精确的检索机制。该论文提出了三个权衡维度：
- 近时性（Recency）：给最近发生的事件更高的分数。
- 重要性（Importance）：区分琐事（如刷牙）和关键事件（如表白），由 LLM 自动评估重要性分值
- 相关性（Relevance）：通过计算查询向量与记忆向量之间的余弦相似度（Cosine Similarity）来确定。

### 涌现性社会行为（Emergent Social Behaviors）
多个 Agent 互动时产生的群体动态
- 信息扩散：一个 Agent 知道要办情人节派对，消息会自动通过对话传播给整个社区。
- 关系形成：Agent 会记住新结识的朋友，并在随后的互动中主动提及之前的对话内容。
- 协同工作：原本只有一人发起的派对，最后会有多名 Agent 自发协调时间、装饰场地并准时出席。


## 突出的深度洞察
- 消融实验（Ablation Study）：论文通过对比证明，缺失“反思”、“规划”或“记忆”中的任何一个，Agent 的可信度都会大幅下降。

### 这种架构有什么局限性？
成本与时延（25 个 Agent 模拟两天花费了数千美元的 Token 费用）；幻觉问题（Agent 有时会虚构记忆或过度美化信息）；以及过度礼貌（受 RLHF 影响，Agent 的对话风格往往过于职业和客套）。
